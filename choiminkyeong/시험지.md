# HTML/CSS 실무형 시험

## 시험 안내

- **시험 시간**: 4시간 ~ 8시간
- **제출 방법**: HTML 파일(들) + CSS 파일(들)
- **평가 기준**: 100점 만점, 60점 이상 통과
- **시험 환경**: 개인 컴퓨터, 선호하는 코드 에디터(VS Code, 메모장 등) 사용 가능

---

## 제공 파일

### 1. 견본 사이트
- **주소**: `112.222.157.156:6200`
- **특징**:
  - 웹 브라우저에서 화면 구조와 동작을 **순수 시각적으로만 확인**할 수 있습니다.
  - 개발자 도구(DevTools) 등 브라우저 기능이 차단되어 있어 **코드 참조는 불가능**합니다.

### 2. 제공 파일
- `img/` 폴더 — 사용 가능한 이미지 파일들
  - img_1.jpg (배경 이미지)
  - item_1.jpg ~ item_5.jpg (카드 이미지)
  - 자유롭게 재사용하거나 다른 이미지로 교체 가능

### 3. 활용 방법
- 견본 사이트(`112.222.157.156:6200`)를 웹 브라우저에서 열어 **화면 구조와 레이아웃을 시각적으로 확인**하세요.
- **스타일이 견본과 정확히 같을 필요는 없습니다.** 전반적인 화면 구성과 기능이 유사하면 정답입니다.

## 구현 요구사항

다음을 만족하는 반응형 웹 사이트를 구현하세요.

### 기본 구조
- [ ] **HTML 기본 문서 구조**
  - DOCTYPE, html lang, head 메타정보(charset, viewport), title, CSS 파일 링크

- [ ] **페이지 섹션 구성**
  - `<header>`, `<main>`, `<footer>` 등 시맨틱 태그로 구성 권장
  - 또는 `<div class="header">`, `<div class="main">`, `<div class="footer">` 등 div + class 속성으로 구현 가능

- [ ] **Header 섹션 구현**
  - 제목(h1 또는 h2)
  - 설명 텍스트
  - 액션 버튼 또는 링크 포함

- [ ] **Main 섹션: 카드 목록**
  - 최소 5개의 카드(card) 요소
  - 각 카드에는:
    - 이미지 또는 배경 이미지
    - 제목(h3 또는 p)
    - 설명 텍스트
    - 버튼 또는 링크

- [ ] **Footer 섹션**
  - 저작권 정보 또는 간단한 텍스트 포함

### 스타일링
- [ ] **CSS 기본 설정**
  - CSS Reset (margin, padding 초기화)
  - `box-sizing: border-box` 적용

- [ ] **전반적인 스타일**
  - 색상, 폰트, 여백을 일관성 있게 적용
  - 페이지 전체적으로 완성도 있는 디자인

- [ ] **버튼 스타일**
  - 기본 상태와 hover 상태 다르게 표현
  - 클릭 가능함이 직관적으로 드러나기

- [ ] **Footer 스타일**
  - 페이지 하단에 적절히 배치

### 레이아웃 (필수)
**다음 중 하나 이상의 방법으로 레이아웃을 구현하세요:**

- [ ] **Flex 또는 Grid 사용**
  - Header 영역을 Flex 또는 Grid로 정렬
  - 카드 목록(Main)을 Flex 또는 Grid로 다단 배치
  - 카드 내부 요소를 Flex로 정렬

**선택 사항 (레이아웃 추가 기법):**
- [ ] **배경 이미지 처리**
  - `background-image`, `background-size: cover` 등 활용
  - 이미지의 가로세로 비율을 유지하며 자동 조정되도록 구현

### 반응형 웹 (선택 사항, 하지만 권장)
- [ ] **화면 크기에 맞춰 레이아웃 자동 조절**
  - 방법 1: Grid의 `auto-fit` 또는 `auto-fill`과 `minmax()`를 활용한 자동 반응형
  - 방법 2: Media Query를 활용한 브레이크포인트 설정
  - 모바일(768px 이하) ← → 데스크톱(1200px 이상)에서 모두 잘 보이도록

### 세부 완성도 (선택 사항)
- [ ] **마우스 호버 효과**
  - 버튼/카드에 hover 스타일 적용

- [ ] **Smooth Scroll**
  - 앵커 링크 클릭 시 부드럽게 스크롤

- [ ] **기타 세부 효과**
  - 트랜지션, 그림자, 둥근 모서리 등 시각적 완성도 향상

## 유의 사항

- **스타일이 견본과 정확히 같을 필요는 없습니다.**
  → 화면 구조와 레이아웃의 개념만 제대로 이해하고 구현되면 정답입니다.

- **Flex와 Grid 중 어느 것을 선택해도 됩니다.**
  → 레이아웃이 의도대로 구현되면 모두 동일하게 평가합니다.

- **미디어쿼리를 사용하지 않아도 됩니다.**
  → Grid의 `auto-fit/auto-fill + minmax()` 패턴으로도 반응형을 구현할 수 있습니다.

- **완벽함보다는 이해가 중요합니다.**
  → 기본 구조를 제대로 구현하는 것이 고급 효과보다 높은 점수입니다.
